    ¶软件开发方法

    • 结构化法 - c语言为主

      • 自顶向下, 逐步分解求精
      • 严格分阶段,阶段产出标准化应变能力差

    • 面向对象方法 - c++, java …

      • 自底向上
      • 阶段界限不明
      • 应变能力更强,更符合人们的思维习惯

    • 面向服务的方法 - 面向对象,但是封装的层级更高

      • 粗粒度,松耦合
      • 标准化和构建化
      • 抽象级别
        • 操作 - 低级
        • 服务 - 中级
        • 业务流程 - 高级

    • 原型法:[需求阶段]

      • 原型法独立于其它三个阶段,是独立的方法
      • 针对需求不明确
      • 按功能:
        • 水平原型(界面)
        • 垂直原型(复杂算法)
      • 按最终结果:
        • 抛弃式原型
        • 演化式原型

    ¶编码及浮点数运算

    • 浮点数表示[1]

      • $$
        N = 尾数 * 基数^{指数(阶码)}
        $$

    • 运算过程:

    • 对阶[2] > 尾数计算[3] > 结果格式化[4]

    • 特点:

      1. 一般尾数用补码[5],阶码[6]用移码[7]
      2. 阶码的位数决定数的表示范围, 位数越多范围越大
      3. 尾数的位数决定书的有效精度,位数越多精度越高
      4. 对阶时,小数向大数看齐
      5. 对阶是通过较小数的尾数右移实现的

    ¶CISC与RISC

    指令系统类型 指令 寻址方式 实现方式 其它
    CISC(复杂) 数量多,使用频率差别大,可变长格式 支持多种 微程序控制技术(微码)[8] 研制周期长
    RISC(精简) 数量少,使用频率接近 ,定长格式,大部分为单周期指令,操作寄存器,只有Load/Store操作内存 支持方式少 增加了通用寄存器;硬布线逻辑控制为主;适合采用流水线 优化编译,有效支持高级语言
    • 复杂指令计算机(CISC, Complex Instruction Set Computers)[9]
    • 精简指令计算机(RISC, Reduced Instruction Set Computers)

    ¶流水线技术

    • 相关参数计算: 流水线执行时间计算, 流水线吞吐率, 流水线加速比
    • 流水线是指在程序执行时多条指令重叠进行操作的一种准并行处理实现技术. 各种部件同时处理是针对不同指令而言的,它们可同时为多条指令的不同部分进行工作,以提高部件的利用率和指令的平均执行速度

    3. 一旦两个操作数的指数对齐,就可以对两个尾数(也称为有效数字)执行加、减、乘或除等算术运算。根据运算的类型,对应的规则会应用于尾数。 ↩︎

    4. 了解到有这个过程就行了,这里不做考察 ↩︎

    5. 补码(Two’s Complement): 补码主要用于表示有符号整数。对于一个N位二进制数,如果它是正数,那它本身就是该数的补码表示;如果它是负数,那么它的补码是其绝对值的反码加1。使用补码可以使加法和减法运算同构。 ↩︎

    6. 在这指的是指数 ↩︎

    7. 移码(Excess-K Code): 移码也是表示浮点数指数部分的一种码制,但与阶码不同。移码是将真实指数值加上一个常数K(通常为2^(n-1)-1,其中n是指数位数)。这种编码方式的优点是,移码的最高有效位总是1,便于判断指数的正负号。 ↩︎
